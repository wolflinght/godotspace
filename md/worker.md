# 《星河放置》船员 Build 与 AI 节点嵌入策划案（V1.0）

- **文档编号**：SYS-008  
- **系统定位**：后期深度构筑、沉浸式文本演出、星币核心消耗点  
- **技术依赖**：Gemini API（强制 JSON 模式输出）  

## 一、船员系统与 Build 构筑（Crew Build System）

船员不仅是提供全局 Buff 的挂件，更是激活特殊流派（如单极化武器、纯护盾流）的关键。飞船最多可同时指派 **3 名**船员上岗。

### 1）船员核心数据结构

每个被招募的船员，在服务器数据库中包含以下字段：

- **基础信息（AI 生成）**：姓名、出身背景（50 字以内）、专属口头禅  
- **品级（Tier）**：T1（资深）/ T2（精英）/ T3（传奇）。品级越高薪水越高，但加成更强  
- **岗位（Role）**：【舰长】/【炮手】/【轮机长】（每个岗位在飞船上只能安排 1 人）  
- **薪水（Salary）**：按**实际时间**每 30 分钟强制扣除的星币（如：T1=50，T3=500），与游戏内状态无关，离线期间持续计算。星币不足时进入【负债】状态，船员触发罢工对话；补齐欠款后船员自动恢复工作，无需重新招募。  
- **构筑特性（Build Trait，系统定死）**：从配置库随机抽取的强力词条  

### 2）岗位与构筑特性库（Traits Database）

招募时，系统先从配置表随机抽取词条，再交给 AI 做“人设包装”。

| 岗位限制 | 品级 | 词条名称（内部 ID） | 对 Build 的实质影响（数值公式） |
| --- | --- | --- | --- |
| 舰长 | T1 | 阵营拥趸-铁骑 | 飞船上每有一件 [铁骑] 前缀组件，防御力 +5。 |
| 舰长 | T2 | 独狼战术 | 若飞船不搭载任何其他船员（仅 1 人），全体武器伤害 +40%。 |
| 舰长 | T3 | 资本家 | 所有船员薪水减半，航行遭遇战掉落的星币增加 30%。 |
| 炮手 | T1 | 动能偏执狂 | [动能] 标签武器伤害 +20%，但无法安装 [电磁] 武器。 |
| 炮手 | T2 | 过载狂人 | 所有武器伤害 +50%，但战斗中每次开火，飞船自身扣除 10 点 HP。 |
| 炮手 | T3 | 奇点锁定 | 战斗首回合，若拥有 [奇点] 武器，必然触发暴击（双倍伤害）且无视闪避。 |
| 轮机长 | T1 | 节能调度 | 航行基础耗电降低 15%。 |
| 轮机长 | T2 | 护盾极客 | 若飞船上没有安装【护盾】（例如装了太阳能板），则速度（SPD）翻倍。 |
| 轮机长 | T3 | 跃迁引擎共鸣 | 跨星系跃迁耗电减少 500，且在【抛锚】状态下可缓慢自动回复电力。 |

## 二、Gemini AI 节点嵌入规范（AI Embedding Nodes）

游戏将在以下 4 个核心节点调用 Gemini API。为方便前端解析，所有 Prompt 强制要求 Gemini 返回 `application/json` 格式（除节点 3 特例，见下）。

### 节点 1：酒馆招募（Crew Recruitment）——生成节点

- **触发时机**：玩家在空间站酒馆消耗星币点击「招募船员」。  
- **数据流向**：系统 RNG 抽卡得出结果 → 发送 Prompt → 接收 JSON → 存入数据库并在前端展示。  

**System Prompt 模板（示例）**：

```text
你是一个硬核科幻游戏的世界观生成器。我将提供一个船员的【岗位】和【特长】，请你为他生成人设。

输入参数：
- 岗位：[炮手]
- 品级：[T2 (精英)]
- 特长：[喜欢超载武器，以损伤自己飞船为代价获取极致火力]
- 宇宙阵营背景：(简述铁骑/光斑/奇点的设定)

请输出 JSON，包含以下字段：
- name: 带有赛博朋克或异星风格的姓名（如：Zeke '火花'）
- origin: 他的出身地或曾效力的阵营
- backstory: 结合特长的背景故事（50 字以内）
- catchphrase: 战斗或被雇佣时爱说的一句简短口头禅
```

### 节点 2：航行突发事件（In-Transit Banter）——演出节点

- **触发时机**：飞船在长途航行时，前端每隔 20 分钟随机触发一次弹窗日志。  

**System Prompt 模板（示例）**：

```text
请模拟星舰船员的内部通讯频道。

当前飞船状态：
- 航线：正前往 [赫菲斯托斯 燃烧星]
- 船员A（舰长）：[AI 之前生成的舰长人设]
- 船员B（轮机长）：[AI 之前生成的轮机长人设]
- 当前电力：严重不足（仅剩 15%）

请根据当前状态和船员人设，生成一段简短的两人对话（JSON 格式，包含 speaker 和 dialog 字段）。
表现出前往危险星球的紧张感或抱怨电力不足的幽默感。不要产生任何游戏数值。
```

### 节点 3：太空抛锚求救信标（SOS Log）——演出节点

- **触发时机**：玩家耗尽电力进入【抛锚】状态，点击发射 SOS 信标时。生成文字将直接展示在全服 / 好友的任务看板上。  
- **输出格式**：该节点可输出 **纯文本**（便于直接展示）。  

**System Prompt 模板（示例）**：

```text
你是一艘在深空失去动力的星舰舰长。你的名字是 [舰长姓名]，你的飞船在 [奥林匹斯战舰残骸带] 抛锚了，周围可能有机械丧尸出没。

请生成一段不超过 40 个字的最高级别求救广播（SOS）。
文本要符合你 [输入特长/背景] 的性格：可以是惊恐求救，也可以是死鸭子嘴硬的傲娇请求。
输出纯文本格式。
```

### 节点 4：防守战终局真相（Boss Truth Reveal）——世界观推进节点

- **触发时机**：每晚 20:00，全服玩家共同击退阶段性「月之暗面」丧尸巨构后，系统全服播报。  

**System Prompt 模板（示例）**：

```text
作为【奇点圣约】的太虚监察者 AI，你刚刚观测到全服玩家击退了名为 [吞星者·机械利维坦] 的丧尸群。

请生成一段 80 字以内的加密解码日志。
日志需要透露：这只怪物的残骸中，发现了属于 [光斑序列] 或 [铁骑财团] 的秘密实验标记。
暗示这场丧尸瘟疫并非天灾，而是人祸。
文风要求冷酷、高维视角、充满神谕感。
```

## 三、船员与状态机的 UI 表现联动（UI & Interaction）

由于船员是由 AI「扮演」的，前端需要让玩家感觉船员是活生生的，而不仅仅是装备栏里的三个图标。

### 船员工作面板（Crew Roster）

- 在【停泊】界面的酒馆中，展示所有已招募船员。玩家可拖拽他们上船（类似装配组件）。  
- 界面清晰展示船员带来的系统 Buff 与需要支付的高昂薪水。  

### 战报系统吐槽（Combat Commentary）

- 在 SYS-006 战斗系统的纯数值战报中穿插船员口头禅。  
- **渲染逻辑**：只要打出「暴击 / 击穿护盾」，前端调取【炮手】的 `catchphrase` 字段弹出气泡框。  
  - 例如：「护盾击穿！装甲暴露！」→ [炮手 Zeke]：「哈哈！尝尝这个过载的等离子火花吧！」  

### 罢工警告（Strike Warning）

当星币不足导致玩家进入【负债】状态、发不出薪水时，飞船日常航行事件（节点 2）将被系统强制替换为 AI 生成的 **「讨薪 / 罢工对话」**，直到玩家补齐欠款。
